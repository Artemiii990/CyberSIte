Quality Metrics & Test Scenarios

Neon-themed summary of CyberSite KPIs and validation scenarios, designed for visual clarity and performance overview.

Quality Metrics

M1 – Availability (SLA)

Target ≥ 99.5% uptime monthly.

Measured via monitoring services (UptimeRobot, StatusPage).

M2 – Page Load Speed

Target < 2.5s for 75th percentile users.

Monitored via Lighthouse & Web Vitals.

M3 – Error Rate

Target < 0.5% 5xx errors per week.

Observed via APM & logs (Sentry).

M4 – User Satisfaction

Target ≥ 4.5 / 5 average rating.

Collected through post‑interaction surveys.

Test Scenarios

T1 – User Registration

Steps: Fill form → Submit → Confirm email. Expected: Success message, user logged in.

T2 – Weak Password Blocking

Steps: Try password “123456”. Expected: Validation error “Password too weak”.

T3 – LCP Measurement

Steps: Run Lighthouse → check median LCP. Expected: < 2.5s load time.

T4 – Profile Data Consistency

Steps: Edit phone → refresh page → check DB/API. Expected: Consistent data.

T5 – XSS Protection

Steps: Submit script in contact form. Expected: Sanitized text, no script execution.

T6 – CI/CD Gate

Steps: Deploy with >0.5% errors. Expected: Release blocked, notification sent.
